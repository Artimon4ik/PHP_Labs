### Задание 1. Анализ HTTP-запросов. Часть 1

В данном задании был проанализирован HTTP-запрос, который отправляется браузером при загрузке страницы  
`https://ru.wikipedia.org/wiki/HTTP`.

![alt text](image.png)


> **URL запроса:**

- https://ru.wikipedia.org/wiki/HTTP  

Данный URL указывает на страницу со статьёй HTTP в русскоязычной версии сайта Wikipedia.



> **Метод запроса:**

`GET`

Метод GET используется для получения данных с сервера.  
При загрузке страницы браузер запрашивает HTML-документ по указанному URL.  
Запрос не изменяет данные на сервере и применяется исключительно для получения информации.



> **Статус ответа:**

`200 OK`  

Статус ответа 200 означает, что запрос был успешно обработан сервером, и запрашиваемая страница была найдена и передана клиенту.


---

> **Заголовки запроса:**

![alt text](image-1.png)

В запросе присутствуют следующие основные заголовки:

- `User-Agent` — содержит информацию о браузере и операционной системе пользователя  
- `Accept` — указывает, какие типы данных может принять клиент  
- `Accept-Language` — задаёт предпочтительный язык содержимого  
- `Accept-Encoding` — определяет способы сжатия данных  
- `Cookie` — используется для хранения сессионных данных  
- `Referer` — указывает страницу, с которой был выполнен переход  


---

> **Заголовки ответа:**

![alt text](image-2.png)

В ответе сервера присутствуют следующие заголовки:

- `Content-Type` — определяет тип возвращаемых данных (HTML)  
- `Content-Encoding` — указывает, что данные передаются в сжатом виде  
- `Content-Length` — размер тела ответа  
- `Cache-Control` — управляет кэшированием данных  
- `Server` — информация о сервере  


> **Тело запроса и ответа:**

Тело запроса отсутствует, так как используется метод GET.  
Тело ответа содержит HTML-код веб-страницы, который используется браузером для отображения содержимого статьи.


**Вывод:**  

В результате анализа можно сделать вывод, что при загрузке страницы Wikipedia используется стандартный HTTP-запрос с методом GET и успешным статусом ответа 200.

### Задание 4. Составление HTTP-запросов

1. Составьте GET-запрос к серверу по адресу http://sandbox.usm.com, указав в заголовке User-Agent ваше имя и фамилию.


>**GET-запрос:**

```http

GET / HTTP/1.1
Host: sandbox.usm.com
User-Agent: Gherganov Artiom

```

> Что такое `User-Agent` и для чего он используется?

- `User-Agent` (агент пользователя) - это короткая текстовая строка, которую браузер или иное приложение (бот, парсер) отправляет на веб-сервер с каждым HTTP-запросом. Она сообщает сайту информацию о браузере, операционной системе и типе устройства, помогая отобразить корректную версию страницы. 


Он используется для:

- идентификации клиента на сервере;
- адаптации контента под устройство или браузер;
- ведения статистики и логирования;
- фильтрации или ограничения доступа.

2. Составьте POST-запрос к серверу по адресу http://sandbox.usm.com/cars, указав в теле запроса следующие параметры:

>**POST-запрос**

```http
POST /cars HTTP/1.1
Host: sandbox.usm.com
Content-Type: application/x-www-form-urlencoded
User-Agent: Gherganov Artiom

make=Toyota&model=Corolla&year=2020
```

- Данный запрос используется для отправки данных на сервер с целью создания нового ресурса (в данном случае - авто).

**Какие ещё методы HTTP-запросов существуют?**

Основные методы HTTP:

- `GET` Метод GET запрашивает представление ресурса. Запросы с использованием этого метода могут только извлекать данные.

- `HEAD` HEAD запрашивает ресурс так же, как и метод GET, но без тела ответа.

- `POST` POST используется для отправки сущностей к определённому ресурсу. Часто вызывает изменение состояния или какие-то побочные эффекты на сервере.

- `PUT` PUT заменяет все текущие представления ресурса данными запроса.

- `DELETE` DELETE удаляет указанный ресурс.

- `CONNECT` CONNECT устанавливает "туннель" к серверу, определённому по ресурсу.

- `OPTIONS` OPTIONS используется для описания параметров соединения с ресурсом.

- `TRACE` TRACE выполняет вызов возвращаемого тестового сообщения с ресурса.

- `PATCH` PATCH используется для частичного изменения ресурса.

3. Составьте PUT-запрос к серверу по адресу http://sandbox.usm.com/cars/1, указав в заголовке User-Agent ваше имя и фамилию, в заголовке Content-Type значение application/json и в теле запроса указать параметры:

```http
PUT /cars/1 HTTP/1.1
Host: sandbox.usm.com
User-Agent: Иван Иванов
Content-Type: application/json

{
  "make": "Toyota",
  "model": "Corolla",
  "year": 2021
}

```

- Данный запрос используется для полного обновления ресурса с идентификатором `1`.

**В чем разница между PUT и PATCH?**

- `PUT` - полностью заменяет ресурс новыми данными. Если какое-либо поле отсутствует, оно может быть удалено.

- `PATCH` - обновляет только те поля, которые указаны в запросе, не затрагивая остальные данные.
